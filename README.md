# 🎴 섯다 — 친구들과 즐기는 웹 섯다 게임

실시간 멀티플레이 섯다 게임. 계정 생성 시 게임머니 1억 지급, 방 만들기/참가(최대 5명), 전원 준비 후 방장이 시작, 판이 자동으로 계속 진행됩니다. 모든 족보(광땡·땡·알리~세륙·끗·망통)와 특수패(땡잡이, 암행어사, 구사/멍텅구리구사 재경기)를 지원합니다.

## 로컬 실행

```bash
npm install
npm start        # http://localhost:3100
npm test         # 족보 단위 테스트 + 봇 시뮬레이션
```

Supabase 설정이 없으면 자동으로 `data/` 폴더의 JSON 파일에 저장됩니다 (개발용).

## Supabase 연동 (사용자 정보 · 잔액 · 게임 결과 저장)

1. [supabase.com](https://supabase.com)에서 프로젝트 생성
2. 대시보드 **SQL Editor**에서 [`supabase.sql`](supabase.sql) 내용 전체를 실행 (테이블 2개 생성)
   - `seotda_users` — 닉네임, 비밀번호 해시, 잔액, 승/패 전적
   - `seotda_game_results` — 판마다 참가자별 승패(`win/lose/draw/replay`)와 금액 증감(`delta`), 족보, 팟 크기, 시각
3. **Project Settings > API**에서 `URL`과 `service_role` 키 복사
4. `.env.example`을 `.env`로 복사하고 값 입력 (배포 환경에서는 환경변수로 설정)

서버 시작 시 `SUPABASE_URL`이 설정되어 있으면 자동으로 Supabase를 사용합니다.

## 배포

⚠️ **Vercel에는 이 서버를 올릴 수 없습니다.** 이 게임은 Socket.IO 기반 실시간 서버라 항상 켜져 있는 WebSocket 연결이 필요한데, Vercel은 서버리스(요청 단위 실행) 방식이라 지속 연결과 인메모리 게임 상태를 유지할 수 없습니다.

대신 **항상 실행되는 Node 서버 호스팅**을 사용하세요. DB는 그대로 Supabase를 쓰면 됩니다:

| 호스팅 | 무료 여부 | 방법 |
|---|---|---|
| **Render** (추천) | 무료 플랜 있음 | New > Web Service > 저장소 연결, Build: `npm install`, Start: `npm start` |
| **Railway** | 소액 크레딧 | 저장소 연결하면 자동 감지 |
| **Fly.io** | 무료 한도 있음 | `fly launch` |

배포 시 환경변수 3개만 설정하면 됩니다: `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, (선택) `PORT`.

> Render 무료 플랜은 15분간 접속이 없으면 잠들었다가 첫 접속 때 깨어나는 데 30초쯤 걸립니다. 친구들과 게임할 때만 쓰는 용도라면 충분합니다.

## 게임 규칙 요약

- 판돈(삥) 1만원, 2장 섯다. 베팅: 삥 / 체크 / 콜 / 따당 / 하프 / 올인 / 다이 (턴당 30초)
- 게임 중 나가기 버튼 = **나가기 예약** (현재 판이 끝나면 자동 퇴장)
- 게임 중 페이지를 닫거나 새로고침하면 **해당 판 패배 처리** 후 자동 퇴장
- 구사(4·9)는 승자가 알리 이하일 때, 멍텅구리구사(4·9 열끗)는 장땡 이하일 때 재경기 (팟 이월)
- 잔액이 판돈보다 적으면 자동으로 방에서 나가집니다
